# サンプル: TaskFlow

架空のタスク管理SaaS「TaskFlow」を題材にした、YGGDRASILの検証用サンプルです。すべてダミーデータであり、実在のプロダクト・企業とは無関係です。

## 内容

- `knowledge/` — ダミーの資料一式。「ガントチャート機能を廃止すべきか？」という論点で3観点（データ・顧客・社内事情）の意見が実際に対立するよう設計してあります
  - データ観点: 利用率は8%と低いが、利用アカウントの解約率は著しく低く収益の26%を占める
  - 顧客観点: 大口顧客はガント必須と明言する一方、「重い・複雑」という不満も多い
  - 社内観点: ビジョンは「シンプルさ」への回帰を掲げるが、過去に刷新が頓挫した経緯と解約額の承認ルールがある
- `decisions/` — 合議を実行すると決定記録が、`decisions/logs/` に議事録（3体の発言全文）が保存されます

## 試し方

リポジトリのルートで以下を実行し、ダミー資料をスケルトンに流し込みます。

```bash
cp -r examples/taskflow/knowledge/ knowledge/
```

その後、Cursorのチャットで `/decide ガントチャート機能を廃止すべきか？` を実行してください。
